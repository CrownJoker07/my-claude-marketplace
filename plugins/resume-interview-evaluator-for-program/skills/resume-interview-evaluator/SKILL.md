# 简历分析与面试评估指南

## 概述

本Skill提供完整的游戏开发工程师面试评估工作流，包括：
- **从PDF简历自动解析关键信息**（姓名、技能、项目等）
- **自动生成技能评估报告** - 分析候选人技术能力和风险点
- **自动生成面试问题清单** - 根据简历内容定制针对性问题
- 面试评分与评估报告输出
- **自动生成面试评估报告 MD 文件**

**报告特点**：
- 从PDF简历自动提取信息，无需手动输入
- 生成两个独立文件：技能评估报告 + 面试问题清单
- 面试评估部分与面试过程记录使用 `---` 分割线分隔
- 项目经历深挖根据简历实际项目生成针对性问题
- 基础能力考察根据候选人技术栈定制问题

## 快速开始

### 方式1：从PDF简历自动生成（推荐）

使用 `analyze_resume.py` 脚本从PDF简历自动提取信息并生成两个文件：

```bash
# 进入技能目录
cd /Users/zhuangzewei/VSpace/Tools/Interview_Assistant/.claude/skills/resume-interview-evaluator

# 基本用法 - 文件生成在PDF所在目录
python3 scripts/analyze_resume.py /path/to/resume.pdf

# 指定输出目录
python3 scripts/analyze_resume.py /path/to/resume.pdf -o ./reports

# 指定候选人姓名（如PDF中无法识别）
python3 scripts/analyze_resume.py /path/to/resume.pdf --name "张三"
```

**输出位置说明**:
- 默认情况下，生成的文件会放在PDF简历所在的目录
- 使用 `-o` 参数可以指定其他输出目录

#### 输出文件

脚本会生成两个Markdown文件：
```
技能评估报告_{姓名}_{日期}.md      - 技能分析、项目评估、优势与风险
面试问题清单_{姓名}_{日期}.md      - 定制化面试问题和评分表
```

#### 依赖安装

```bash
# 推荐安装 pdfplumber（提取效果更好）
pip install pdfplumber

# 或安装 PyPDF2
pip install PyPDF2
```

### 方式2：面试后生成评估报告

使用 `generate_report.py` 在面试完成后生成结构化评估报告：

```bash
# 交互模式（推荐）
python3 scripts/generate_report.py

# 快速模式
python3 scripts/generate_report.py --name "刘洧瀚" --position "Unity实习生" --score 82

# 完整参数模式（支持项目信息）
python3 scripts/generate_report.py \
    --name "刘洧瀚" \
    --position "Unity游戏开发实习生" \
    --interviewer "面试官姓名" \
    --technical 85 \
    --project 80 \
    --algorithm 75 \
    --teamwork 80 \
    --potential 85 \
    --culture 80 \
    --highlights "Unity技术扎实" "担任主程经验" "获奖项目" \
    --risks "分布式系统描述存疑" "C++能力待验证" \
    --recommendation "建议录用为Unity实习生，有转正机会，可长期培养" \
    --project-name "梦迴/补天" \
    --project-role "客户端开发主程序" \
    --tech-stack "Unity" "Wwise" "UniTask" "行为树" "战斗系统" \
    --weak-areas "C++语法了解" "分布式系统描述" \
    --output-dir "./reports"
```

#### 输出文件

脚本会在指定目录（默认为当前目录）生成如下格式的文件：
```
interview_report_候选人姓名_YYYYMMDD.md
```

## 面试流程模板

总时长约 **30-35分钟**

### 1. 自我介绍（2分钟）

**考察目标**：了解候选人职业规划、求职动机、自我认知

**核心问题**：
- [ ] 请简单介绍一下自己
- [ ] 为什么来面试这个岗位？（观察是否有清晰的职业规划）
- [ ] 为什么选择我们公司/项目组？

**评估要点**：
| 维度 | 优秀表现 | 需关注 |
|------|---------|--------|
| 职业规划 | 有清晰短期+长期目标 | 目标模糊或随大流 |
| 岗位认知 | 了解岗位具体职责 | 对岗位理解偏差 |
| 表达能力 | 逻辑清晰，重点突出 | 过于冗长或过于简短 |

---

### 2. 项目经历深度挖掘（20分钟）

**考察目标**：验证技术深度、项目真实性、问题解决能力

根据候选人简历中的实际项目，生成针对性的面试问题，例如：

**《梦迴》《补天》项目示例问题**：

**项目概述**：
- 请介绍一下《梦迴》《补天》项目的整体架构
- 你担任客户端开发主程序，团队规模？开发周期？
- 代码贡献率80%+是如何统计的？

**音频系统（Wwise）**：
- Wwise与Unity音频系统的核心区别？
- 如何在代码中管理音频事件和状态？

**战斗系统设计**：
- 请描述技能系统的架构设计
- 如何处理技能之间的打断、连携、Buff/Debuff关系？

**技术深度追问**：
- 项目中遇到的最大技术挑战是什么？如何解决的？
- 如果重新设计这个项目，你会做哪些改进？

**评估要点**：
| 维度 | 权重 | 优秀表现 |
|------|------|---------|
| 技术深度 | 30% | 能讲清楚原理和实现细节 |
| 项目真实性 | 25% | 细节一致，经得起追问 |
| 问题解决 | 25% | 有具体案例，思路清晰 |
| 技术视野 | 20% | 了解技术选型的trade-off |

---

### 3. 基础能力考察（5分钟）

**考察目标**：根据候选人技术栈，有针对性地验证计算机基础和编程能力

**针对性问题示例**（根据简历技术栈自动生成）：

**异步编程（UniTask）**：
- UniTask和Unity传统Coroutine的区别？
- async/await的原理是什么？在什么场景下使用？

**薄弱环节补充考察**：
- C++中指针和引用的区别？（针对C++仅了解语法的候选人）
- 请具体说明简历中提到的"分布式系统"是如何实现的？

**必问基础问题**：
- C#中值类型和引用类型的区别？什么是装箱拆箱？
- 什么是GC？如何避免GC Alloc？
- 项目中使用了哪些设计模式？单例模式的优缺点？
- 解释A*寻路算法的原理
- 如何实现一个对象池？有什么好处？

---

### 4. 非技术素质考察（2分钟）

**考察目标**：团队协作能力、抗压能力、学习能力

- [ ] 项目与他人合作有没有遇到过矛盾？如何处理？
- [ ] 合理安排需求的情况下，没做完的需求会如何处理？
- [ ] 工作过程中遇到不懂的问题会如何解决？假设工作日中也没解决，会怎么办？

**评估要点**：
| 维度 | 优秀表现 | 需关注 |
|------|---------|--------|
| 协作能力 | 能妥协，会沟通 | 过于固执或推卸责任 |
| 抗压能力 | 积极面对，主动求助 | 被动等待或消极应对 |
| 学习能力 | 有系统学习方法 | 无章法，纯试错 |

---

### 5. 候选人提问（3分钟）

- [ ] 给候选人提问机会
- [ ] 观察候选人对岗位/团队/项目的关注程度

## 面试评估报告模板

### 评分表

| 评估维度 | 权重 | 得分 | 说明 |
|---------|------|------|------|
| 技术能力 | 35% | /100 | Unity/C#/架构设计 |
| 项目经验 | 25% | /100 | 项目深度/复杂度/贡献 |
| 算法基础 | 15% | /100 | 数据结构/算法/设计模式 |
| 团队协作 | 10% | /100 | 沟通/协作意识 |
| 发展潜力 | 10% | /100 | 学习能力/技术视野 |
| 文化匹配 | 5% | /100 | 价值观/工作态度 |
| **总分** | **100%** | **/100** | |

### 推荐等级

- **A级（强烈推荐）**：总分≥85，无明显短板
- **B级（推荐）**：总分70-84，有培养潜力
- **C级（谨慎考虑）**：总分60-69，有明显风险点
- **D级（不推荐）**：总分<60，不适合该岗位

## 脚本使用说明

### analyze_resume.py

该脚本用于从PDF简历自动提取信息并生成技能评估报告和面试问题清单。

**功能**：
- 自动提取PDF文本（支持 pdfplumber 或 PyPDF2）
- 解析关键信息：姓名、岗位、技能、项目经历等
- 分析技能熟练度和风险点
- 生成技能评估报告（技能概览、项目分析、综合评价）
- 生成定制化面试问题清单（针对简历内容）

**参数说明**：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `pdf_path` | PDF简历文件路径（必填） | - |
| `-o, --output-dir` | 输出目录 | PDF所在目录 |
| `--name` | 候选人姓名（覆盖PDF识别结果） | 自动识别 |

**输出文件命名规则**：
```
技能评估报告_{候选人姓名}_{YYYYMMDD}.md
面试问题清单_{候选人姓名}_{YYYYMMDD}.md
```

**示例**：
```
技能评估报告_啊源_20260227.md
面试问题清单_啊源_20260227.md
```

**报告内容特点**：
- 自动识别技术栈（编程语言、游戏引擎、专业技能、工具）
- 根据项目数量评估技能熟练度
- 生成优势亮点和风险点分析
- 项目复杂度评估和风险标注

### generate_report.py

| 参数 | 说明 | 默认值 |
|------|------|--------|

| `--algorithm` | 算法基础得分 | 70 |
| `--teamwork` | 团队协作得分 | 70 |
| `--potential` | 发展潜力得分 | 70 |
